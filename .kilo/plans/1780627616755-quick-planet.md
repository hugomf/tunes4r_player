# Implementation Plan for tunes4r_player

Based on review of FIRST_ASSESMENT.md and current codebase, here's what's implemented vs missing:

## Already Implemented ✅

### Bug Fixes:
- **BUG-1 (YouTube seek re-downloads)**: Header caching logic implemented in `youtube.rs` with TeeReader/ChainReader
- **BUG-2 (Live radio backward seek byte-offset)**: Fixed in `live.rs` with correct formula
- **BUG-3 (Live radio canSeek always false)**: Fixed in `commands.rs::play_live()` - now sets `self.source`
- **BUG-7 (Duplicate log line)**: Fixed in `handling.rs` - only one "[stream] Connected! Detecting format..." line

### Platform Consolidation:
- Created `platform.rs` with basic structure
- Header caching logic in `youtube.rs` shows proper TeeReader/ChainReader implementation

### Architecture Improvements:
- **ARCH-4 (Split decode_and_play_from_read into phases)**: Already implemented - function clearly separated into 8 phases (probe, device init, decoder creation, prebuffer, stream build, spectrum analysis, playback loop, drain)

## Still Missing ❌

### Bug Fixes Needed:
- **BUG-4 (YouTube seek re-resolves manifest)**: Legacy `play_adaptive_buffer_internal`/`play_stream_internal` paths still exist and are used
- **BUG-5 (fast_forward_stream_seek codec corruption)**: Still present in code - needs removal after BUG-1 fix
- **BUG-6 (Flutter live seek slider)**: Flutter-side fix needed (outside Rust scope)
- **MISSING-1 (Live stream buffer poller)**: Need to implement proper buffer display using `start_time`

### Platform Consolidation Needed:
- Remove `ios_file_decoder.rs` and `android_file_decoder.rs`
- Create `decoder/platform/android.rs` (AndroidFetcher only)
- Create `decoder/platform/desktop.rs` (BlockingFetcher)
- Unify `play_file_internal` and `play_stream_internal` via HttpFetcher trait
- Update `decoder/mod.rs` for platform routing
- Gut `android_file_decoder.rs` to only platform_init + AndroidFetcher

### Architecture Improvements Needed:
- **ARCH-1 (Workspace crate split)**: Split into core, youtube, ffi, tui crates
- **ARCH-2 (Retire deprecated functions)**: Remove `play_adaptive_buffer_internal` and `play_stream_internal`
- **ARCH-3 (Replace seek_target_ms with channel)**: Replace Arc<AtomicU64> with mpsc::Sender<SeekCommand>
- **ARCH-5 (DRY YouTube headers)**: Extract header creation to `youtube_request_headers()` helper
- **ARCH-6 (Flutter BufferedSlider)**: Extract widget with explicit isLive parameter

## Priority Order for Implementation:

1. **Complete Bug Fixes** (do first):
   - Fix BUG-4 by migrating all callers to play_pipeline
   - Remove fast_forward_stream_seek (BUG-5) after verifying BUG-1 works
   - Implement MISSING-1 buffer display fix using start_time

2. **Platform Consolidation** (do second):
   - Create platform-specific fetcher implementations
   - Unify decoder functions via HttpFetcher trait
   - Remove duplicate platform files

3. **Architecture Improvements** (do last, after stability):
   - Workspace crate split
